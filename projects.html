Pit Scheduler

Private team repo

A Google Apps Script and Google Sheets scheduler for robotics teams that posts shift schedules to Slack and sends individual reminders.

Problem

During robotics competitions, team members can get caught up in the rush of the event and lose track of when their pit shift ends. Pit Scheduler was built to make shift timing clear, remind people when they need to rotate out, and keep the pit organized without someone manually chasing everyone down.

My Role

Built the scheduler interface and Slack reminder automation for my robotics team.

Tech Stack

Google Apps Script, Google Sheets, HTML, CSS, JavaScript, Slack API.

Features

Shift CRUD, CSV bulk import, Slack channel posts, DM reminders, printable schedules, and live on-duty panels.

Challenge

Coordinating spreadsheet data, time-based triggers, Slack user mapping, and a simple web UI in one Apps Script project.

Lesson Learned

Operational tools need clear data validation, dependable notifications, and workflows that are quick during busy events.

MECO Mission Control Mobile

GitHub

An Expo and React Native mobile client for robotics manufacturing, planning, meetings, task visibility, and operations workflows.

Vision

MECO Mission Control Mobile is being built because our robotics team has tried many tools that worked, but not in one shared place and not exactly the way our team needed. The goal is to create one team operations app for everyone, from highly involved students to mentors who only need to help with specific planning tasks.

My Role

Building mobile app flows and learning how to connect the experience to server-backed authentication and role-based features.

Tech Stack

Expo, React Native, React, TypeScript, ESLint, EAS release automation.

Features

Subsystem dashboard, task workflow states, meeting RSVP and attendance, work-log visibility, manufacturing queues, and QA outcomes.

Current Status

Core flows that do not depend on login are working. Google SSO and role-based views for groups like programming and mechanical are planned next.

Challenge

Building my first project that needs a server, authentication, and role-aware behavior across different types of users.

Lesson Learned

I learned how mobile apps connect to server-backed features and how to approach Google SSO implementation with AI as a learning partner.
